# Kodia CLI Usage Guide

The Kodia CLI is a powerful developer tool designed to accelerate fullstack development with Golang Gin and SvelteKit. Inspired by frameworks like Ruby on Rails and Laravel, it provides advanced scaffolding, a hot-reloading development server, and an interactive Go REPL.

## 🛠️ Scaffolding Commands

### generate crud
Generates a complete vertical slice of a feature, including backend and frontend layers.

```bash
kodia generate crud [Name] --fields=[fields] [flags]
```

**Example:**
```bash
kodia generate crud Product --fields=name:string,description:text,price:decimal,stock:int --with-migrations
```

#### Field Parsing
Fields are defined as `name:type:options`.
- **Types**: `string`, `text`, `int`, `bigint`, `float`, `boolean`, `timestamp`, `references`, `enum`.
- **Options**: `unique`, `nullable`, or any custom validation tag (e.g., `email`, `min=8`).

**Relationships:**
Use the `references` type to automatically create foreign keys.
```bash
kodia generate crud Post --fields=title:string,author:references:User
```
This generates an `author_id` field in Go and SQL, and automatically adds a foreign key constraint to the `users` table.

### generate model
Generate only the domain entity and associated model logic.
```bash
kodia generate model Product --fields=name:string,price:float
```

### generate policy | request | resource
Generate specialized layers for authorization, DTOs, or API transformers.
```bash
kodia generate policy Product
kodia generate request StoreProductRequest --fields=name:string,price:float
kodia generate resource ProductResource --fields=name:string,price:float
```

## 🚀 Development Server

### serve --watch
Start the development server with auto-reload capabilities.
```bash
kodia serve --watch
```
- Monitors `backend/` and `frontend/` directories.
- Automatically recompiles and restarts the server on file changes (`.go`, `.html`, `.tmpl`, `.yaml`).
- Features intelligent debouncing to prevent excessive reloads.

## 🔮 Interactive REPL

### tinker
Enter an interactive Go REPL pre-loaded with your application environment.
```bash
kodia tinker
```
- Test domain logic and snippets instantly.
- Powered by `yaegi` for high-performance interactive Go execution.

## 🏗️ Architecture Integration

Scaffolded features are automatically wired into:
1. **Dependency Injection**: Registered in `backend/cmd/server/main.go`.
2. **Routing**: Registered in `backend/internal/adapters/http/router.go`.
3. **Ports**: Interfaces generated in `backend/internal/core/ports/`.
